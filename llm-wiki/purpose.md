# 研究目的與方向

## 核心目標

把 finerenone / 非類固醇型礦皮質素受體拮抗劑（nsMRA）七大主題的深度文獻回顧，編譯成一個**可持續累積、互相連結**的知識庫，服務兩種使用場景：

1. **臨床決策與衛教**——內分泌科／腎臟科醫師在門診要回答「這個病人該不該加 finerenone、什麼時候加、怎麼監測」時，能直接查到有證據等級標示的答案。
2. **演講與投稿素材庫**——製作進階演講（聽眾已熟悉 FIDELIO/FIGARO/FIND-CKD 基本結果）或撰寫 review 時，能快速調出機轉、次分析、方法學爭議與尚待驗證的假說。

知識庫繼承母專案的硬規則：**所有作者、年份、DOI、樣本數、HR/OR 都必須能回溯到本地全文語料**。實體頁與主題頁只做組織與改寫，不新增原始語料沒有的數字。

## 關鍵問題

1. **機轉層**：finerenone 與 spironolactone 的差別，究竟有多少來自受體選擇性／解離動力學（residence time、off-rate），有多少只是安全性剖面不同？合成端抑制（ASI：baxdrostat / lorundrostat / vicadrostat）與受體端拮抗，臨床結局會不會一致？
2. **替代終點層**：UACR 下降是「伴隨型生物標記」還是「機轉連結的替代終點」？UACR 與收縮壓各中介了多少心血管益處？停藥後 UACR 反彈說明了什麼？
3. **加藥策略層**：RASi + SGLT2i 之後，finerenone 該序列加藥還是同步起始？CONFIDENCE 的 over-additive 效應該怎麼解讀，停藥後還剩多少？
4. **族群層**：誰該「最先治療」（絕對事件率高者）而非誰「相對益處最大」？進階 CKD、亞洲族群、HF/LVH/胰島素阻抗各自的效果修飾證據強度如何？
5. **安全工程層**：高血鉀與 eGFR dip 該用什麼場景別的監測流程管理？低血鉀↓54% 是否中介了心血管益處？CYP3A4 交互作用與 K binder 的實際處方規則是什麼？
6. **版圖擴張層**：T1D（FINE-ONE）、非糖尿病 CKD（FIND-CKD）、HFpEF/HFmrEF（FINEARTS-HF）、pooled（FINE-HEART）之後，nsMRA 在 CKM 平台上的定位是什麼？哪些是已核准、哪些只是可外推、哪些仍是假說？

## 研究範圍

**涵蓋：**
- finerenone 的受體藥理、PK/PD、BP-independent 心腎保護機轉
- 白蛋白尿與 eGFR slope 作為替代終點的方法學（含中介分析、資格化標準）
- 四大支柱／早期聯用的加藥策略與 CONFIDENCE 三臂設計
- 次族群分析與效果修飾的方法學（相對 vs 絕對益處、NNT/NNH）
- 高血鉀、eGFR dip、低血鉀、CYP3A4、K binder 的安全管理與門診 workflow
- 超越 T2D 的適應症版圖：T1D、非糖尿病 CKD、glomerular 亞群、HF
- 醛固酮合成酶抑制劑（ASI）與 nsMRA 的機轉對比，含原發性醛固酮症（PA）應用
- 相關統計迷思與深度解讀（每個主題各有一份）

**不涵蓋：**
- 類固醇型 MRA（spironolactone / eplerenone）本身的完整臨床回顧——只在與 finerenone 對比時出現
- 一般糖尿病或 CKD 的基礎照護（血糖、血壓、血脂目標值）
- 藥價、給付與健保申報規則
- 流程稽核檔（`_audit_*.md`、`MISSING_FULLTEXT.md`、`撰寫方法論.md`）——這些留在母專案，不進知識庫
